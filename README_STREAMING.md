

## 📁 文件结构

```
core/
├── stream_processor.py    # 流式处理器核心逻辑
io_utils/
├── db_handler.py         # 数据库处理器（已优化）
main.py                   # 主程序（已优化）
```

## 🔧 使用方法

### 普通模式（推荐）
```bash
python main.py
```

**特点**：
- 逐文件处理，避免内存堆积
- 默认批处理大小：1000条/批
- 内存阈值：5000条时触发垃圾回收
- 适合大多数场景

### 大文件模式（超大Excel）
```bash
python -c "from main import main_for_large_files; main_for_large_files()"
```

**特点**：
- 分块读取Excel（1万行/块）
- 更大的批处理大小（2000条/批）
- 适合50万行以上的超大文件

## 🎯 性能优化点

### 1. 内存管理
- **逐文件处理**：处理完一个文件就释放内存
- **批量入库**：每1000条记录入库一次
- **强制GC**：定期触发垃圾回收

### 2. 数据库优化
- **连接复用**：减少连接创建开销
- **批量提交**：提高写入效率
- **错误处理**：支持事务回滚

### 3. 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `BATCH_SIZE` | 1000 | 每批入库的记录数 |
| `MEMORY_THRESHOLD` | 5000 | 内存阈值，触发垃圾回收 |
| `chunk_size` | 10000 | 大文件分块大小 |
